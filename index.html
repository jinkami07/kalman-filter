カルマンフィルタ 解説

🔍カルマンフィルタとは

カルマンフィルタ（KF）は「物理モデルによる予測」と「センサ観測による補正」を 交互に繰り返すことで，ノイズの多い環境でも状態を最適に推定するアルゴリズムです。

システムが線形かつノイズがガウス分布に従う場合， KF は最小分散推定（MMSE）の意味で最適な推定器になります。

不確かさの大きさは共分散行列 $P_k$ で管理されます。 $P_k$ が小さいほど推定値への信頼度が高く，カルマンゲイン $K_k$ も小さくなって センサ情報の重みが下がります。

イノベーション（新情報）$y_k = z_k - H\hat{\mathbf{x}}^-_k$ は 「モデル予測と実観測のズレ」です。これが大きいほど推定を大きく修正します。

⚙️なぜこの問題に必要か

ロボット関節の PD 制御では微分項 $K_d \dot{\theta}$ のために角速度 $\omega$ が必要です。 しかし一般的な関節エンコーダが測れるのは角度 $\theta$ のみです。

素朴な解決策：数値微分
$\hat{\omega}_k \approx (\theta_k - \theta_{k-1}) / \Delta t$
→ 観測ノイズが微分で増幅され，高周波ノイズが大きくなる問題がある。

カルマンフィルタは運動方程式（$A$ 行列）を内部モデルとして持ち， 角度の観測列から角速度を間接的かつ最適に推定します。 上の「角速度グラフ」で，観測なしに真値を追跡できていることを確認してください。

これが KF の最大の強みの一つ：観測できない状態量の推定（オブザーバとしての機能）

📐状態空間モデルの導出

関節の運動方程式（連続時間）：

連続時間モデル
$$J\dot{\omega} = \tau - b\omega \;\Longrightarrow\; \dot{\omega} = \frac{\tau}{J} - \frac{b}{J}\omega$$ $$\dot{\theta} = \omega$$

前進オイラー法で離散化（サンプリング周期 $\Delta t$）：

離散時間モデル
$$\theta_{k+1} = \theta_k + \Delta t\,\omega_k$$ $$\omega_{k+1} = \!\left(1 - \frac{b\Delta t}{J}\right)\omega_k + \frac{\Delta t}{J}\tau_k$$

これを行列形式にまとめると $\mathbf{x}_{k+1} = A\mathbf{x}_k + B\tau_k$ となります。 $A$ の $(2,2)$ 要素 $1 - b\Delta t/J$ は粘性減衰を表し，$1$ に近いほど速度が持続しやすくなります。

🔄予測ステップと更新ステップの直感

① 予測（Predict） — 物理モデルを信じて次の状態を先読みする。

$\hat{\mathbf{x}}^-_k = A\hat{\mathbf{x}}_{k-1} + Bu_k$ : モデルで1ステップ先を計算
$P^-_k = AP_{k-1}A^\top + Q$ : モデルが持ち込む不確かさ $Q$ を加算 → $P$ が増大

② 更新（Update） — センサ観測で予測を補正する。

$K_k = P^-_k H^\top (HP^-_kH^\top + R)^{-1}$ : 「モデル不確かさ vs センサ不確かさ」の比
$\hat{\mathbf{x}}_k = \hat{\mathbf{x}}^-_k + K_k \underbrace{(z_k - H\hat{\mathbf{x}}^-_k)}_{\text{イノベーション}}$ : 予測をイノベーション分だけ修正
$P_k = (I - K_kH)P^-_k$ : センサを使った分だけ不確かさが減少

カルマンゲイン $K_k$ の意味：
$K_k \to 0$ のとき $\hat{\mathbf{x}}_k \approx \hat{\mathbf{x}}^-_k$ （センサを無視，モデルのみ信頼）
$K_k \to 1$ のとき $\hat{\mathbf{x}}_k \approx z_k / H$ （センサを全信頼，モデルを無視）

$K_k$ は $P_k$ が収束すると定常値に落ち着きます（上のゲイングラフで確認できます）。 定常カルマンゲインを事前計算して固定する手法を 定常カルマンフィルタ と呼びます。

🎛️パラメータの意味とチューニング指針

パラメータ 意味 大きくすると
$R$ 観測ノイズ分散
（センサの不確かさ）		 $K$ が小さくなり，センサより
モデルを信頼 → 推定が滑らか
$Q_\theta$ 角度プロセスノイズ分散
（角度モデルの不確かさ）		 角度の $P$ が増大し，$K$ が大きく
なって角度の補正が大きくなる
$Q_\omega$ 角速度プロセスノイズ分散
（角速度モデルの不確かさ）		 角速度推定の追従が速くなるが
ノイズも拾いやすくなる
$P_0$ 初期推定誤差分散 立ち上がり時に $K$ が大きく
なる（すぐ収束するので影響小）
チューニングの出発点
$R$ はエンコーダのスペックシートから設定可能（$R \approx \sigma_{\rm sensor}^2$）。
$Q$ はモデル化誤差や外乱の大きさに応じて調整する「設計パラメータ」です。
一般に $R/Q$ の比が重要で，$R \gg Q$ ならモデル重視，$R \ll Q$ ならセンサ重視になります。
実装上の注意
$P_k$ が負定値にならないよう数値的安定性に注意が必要です。 実用では Joseph 形式 $P_k = (I-K_kH)P^-_k(I-K_kH)^\top + K_kRK_k^\top$ や 平方根フィルタが使われることがあります。
